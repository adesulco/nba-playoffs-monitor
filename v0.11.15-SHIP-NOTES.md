# v0.11.15 — SWR cache port to EPL / F1 / Tennis

**Date:** 2026-04-23
**Branch:** main
**Risk:** LOW. Mechanical port of v0.11.7's NBA SWR pattern to six more hooks. Cold-path behaviour unchanged; only the return-visit path gains instant paint.

---

## What shipped

Six hooks now hydrate from `swrCache` on mount instead of starting at empty state:

| Hook | Cache key | TTL |
|---|---|---|
| `useEPLStandings` | `epl-standings` | 5 min |
| `useEPLFixtures` | `epl-fixtures-{back}-{fwd}` | 2 min |
| `useF1Standings` | `f1-standings-{SEASON}` | 15 min |
| `useF1Schedule` | `f1-schedule-{SEASON}` | 24 h |
| `useTennisRankings` | `tennis-rankings-{tour}` | 6 h |
| `useTennisSlamOdds` | `tennis-slam-odds-{slug}` | 15 min |

Pattern is identical to v0.11.7's `usePlayoffData`: `useState(() => readCache(key) ?? fallback)` on mount; `writeCache(key, payload)` on every successful fetch. TTLs tuned per endpoint's natural refresh cadence — tight for live fixtures, wide for the F1 calendar.

### User-visible effect

Return visits to `/premier-league-2025-26`, `/formula-1-2026`, `/tennis` now paint the standings / fixtures / rankings tables on the first frame, then refresh in the background. Matches the NBA dashboard's v0.11.7 behaviour — no more blank-cream-for-2-seconds bounce for a user who came back an hour later.

Cache entries live under `gibol:swr:v1:*` — same namespace as NBA, same schema-version prefix, same quota-safe write path. Across all hooks, a heavy user's cache consumes ~30–60 KB of localStorage.

## Verify after deploy

1. **EPL return visit.** Open `/premier-league-2025-26` once; close tab; reopen. Table + fixture strip should paint without visible load state. DevTools → Application → Local Storage → look for `gibol:swr:v1:epl-standings` and `gibol:swr:v1:epl-fixtures-7-7`.
2. **F1 calendar.** Open `/formula-1-2026`. Expect `gibol:swr:v1:f1-schedule-2026` and `gibol:swr:v1:f1-standings-2026` keys.
3. **Tennis rankings.** Navigate between `/tennis/rankings/atp` and `/tennis/rankings/wta`. First visit each writes; subsequent visits paint instantly.
4. **Stale-but-fresh logic.** Kill the network (DevTools Offline). Reload the EPL page — table paints from cache, then `setError` reflects the failed refresh.
