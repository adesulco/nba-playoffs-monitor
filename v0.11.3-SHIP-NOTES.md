# v0.11.3 ‚Äî Header persists across routes + no more width jolt

**Date:** 2026-04-23
**Branch:** main
**Risk:** LOW. No API, no schema, no data path. Pure chrome-routing refactor on top of v0.11.2's universal masthead.

---

## What shipped

### 1. V2TopBar renders once, persists across route changes

Before: each page rendered `<V2TopBar />` itself. Clicking Home ‚Üí NBA ‚Üí
F1 ‚Üí Tennis re-mounted the masthead on every nav, and while the
destination chunk streamed, `<Suspense fallback>` rendered a
full-viewport blank ‚Äî so the header disappeared too. It felt like a
full page reload.

After: `V2TopBar` is rendered exactly once in `App.jsx`, _above_
`<Suspense>`:

```jsx
<BrowserRouter>
  <V2TopBar />
  <Suspense fallback={<RouteFallback />}>
    <Routes>‚Ä¶</Routes>
  </Suspense>
</BrowserRouter>
```

Route changes now swap only the body. The masthead, its ‚åòK listener,
and its sub-row never unmount.

### 2. Pickers push sub-row content via a tiny store

Pages that need content under the nav (NBA TeamPicker, F1
ConstructorPicker, EPL ClubPicker, Tennis PlayerPicker) used to pass
JSX as `children` to `V2TopBar`. Since `V2TopBar` now lives at the
App root, pages push content via a subscription store:

```js
// src/lib/topbarSubrow.js
import { useSyncExternalStore } from 'react';
let current = null;
const listeners = new Set();
export function setTopbarSubrow(node) { current = node ?? null; listeners.forEach((fn) => fn()); }
export function useTopbarSubrow() { return useSyncExternalStore(subscribe, getSnapshot, getSnapshot); }
```

Pages call `setTopbarSubrow(<Picker/>)` inside `useEffect` and return
a `setTopbarSubrow(null)` cleanup. Only `V2TopBar` subscribes ‚Äî no
cascading context re-renders into every `useApp()` consumer.

### 3. `scrollbar-gutter: stable`

Added to `<html>` in `src/index.css` so the viewport reserves the
scrollbar gutter at all times. Navigating from a short page (Home, no
scroll) to a long one (NBA, scrolls) no longer jolts the content
~15px left as the vertical scrollbar appears.

### 4. Suspense fallback no longer blanks the full viewport

Updated `RouteFallback` in `App.jsx` to `calc(100vh - 48px)` so it
sits under the persistent header. During chunk streams the user now
sees the masthead + a short blank ‚Äî never a full blank page.
