**Day 1 — accessibility foundation (5 of the 6 audit P0s closed)**

- **GIB-001 palette lift.** `--ink-3` and `--ink-4` tokens darkened so muted body-text labels clear WCAG 1.4.3 AA:
  - Dark: `--ink-3 #7388A5 → #9FB4CC`, `--ink-4 #4A5D7A → #6C85A8`. Both ≥ 4.6:1 on `--bg`.
  - Light: `--ink-4 #9A9A9A → #6E6E6E` (5.0:1 on cream). `--ink-3` was already passing on cream.
  - One-token change targets the 123 axe-core contrast nodes the live audit found.
- **GIB-004 focus ring.** Global `:focus-visible` rule expanded with explicit selectors for `a`, `button`, `input`, `[role="button"]`, `[role="option"]`, `[role="link"]`, `[tabindex]`, plus `!important` on `outline` and `outline-offset` so inline style resets can't override. Color switched from `var(--amber)` to `var(--blue)` to give the ring a stronger 3:1 non-text contrast against dark and cream backgrounds. Keyboard users now see where they are on every hub.
- **GIB-002 two `<main>` on every page.** `App.jsx` wraps `<Routes>` in `<main id="main" tabIndex={-1}>` for the skip-link target. **12 page components also shipped their own inner `<main>`** — About, Bracket, BracketEdit, BracketNew, BracketShare, Glossary, HomeV1, Leaderboard, LeaderboardLeague, LeagueJoin, LeagueNew, Login. All 12 demoted to `<section>` so each page now has exactly one landmark. Skip-link behaviour preserved.
- **GIB-016 `aria-current="page"`** on the active nav Link in `V2TopBar.jsx`. Screen-reader rotor now announces which section the user is on.
- **GIB-017 `aria-pressed`** on the BI/EN lang toggle in `V2TopBar.jsx`. Reflects whether BI is currently active.
