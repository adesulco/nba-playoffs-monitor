# v0.11.16 — Tennis title-odds row mobile fix

**Date:** 2026-04-23
**Risk:** LOW. One grid-template tweak + one media-query rule. No data or layout change on desktop.

---

## What shipped

Tennis `/tennis` hero panel renders a "Title odds · Roland Garros" table with one row per player. The row was a 4-column grid: `minmax(84px, 1.4fr) 1fr 40px 30px` (name · progress bar · pct · delta).

At 375 px mobile:
- Before: name column ~141 px — "Alexander Zverev" + country code ellipses into "Alexand…".
- After: name column ~155 px (ratio bumped to 2fr, floor raised to 110 px), and at ≤480 px the progress bar column is dropped entirely so name + pct + delta share the full width without squeezing.

Desktop behaviour unchanged (4-column layout preserved). Tablet (481–720 px) gets the same 4-column grid with the relaxed ratios, so even medium-narrow viewports read cleanly.

### Implementation

- Inline `gridTemplateColumns: 'minmax(110px, 2fr) minmax(48px, 1fr) 40px 30px'` in `Tennis.jsx`.
- Added `className="tennis-odds-row"` on the row wrapper and `className="tennis-odds-bar"` on the progress bar.
- New CSS in `src/index.css`:
  ```css
  @media (max-width: 480px) {
    .tennis-odds-row { grid-template-columns: 1fr 44px 28px !important; gap: 6px !important; }
    .tennis-odds-bar { display: none !important; }
  }
  ```

## Verify after deploy

1. **Desktop** `/tennis` — title-odds panel unchanged; 4-column rows with progress bars.
2. **Tablet** (481–720 px in DevTools device mode) — same 4-column rows, slightly more name-column width.
3. **Mobile** (≤480 px) — progress bar column disappears; name · pct · delta share the row. Long names ("Alexander Zverev", "Jannik Sinner", "Iga Świątek") render without ellipsis.
