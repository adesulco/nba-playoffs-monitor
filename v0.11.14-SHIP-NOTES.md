# v0.11.14 — ⌘K onboarding tooltip (Sprint 6)

**Date:** 2026-04-23
**Branch:** main
**Risk:** LOW. One new component, one CSS block, one mount point. Renders only for first-visit users; localStorage gates re-appearance. No data-path or layout change.

---

## What shipped

Persistent first-visit hint that points at the `⌘K` search pill on desktop, or the bottom-nav search icon on mobile. Copy:

- ID: "Tekan ⌘K untuk cari tim, laga, pemain" / "Ketuk 🔍 untuk cari tim & laga"
- EN: "Press ⌘K to search teams, matches, players" / "Tap 🔍 to search teams & matches"

Shows 1500 ms after first paint. Auto-dismisses on user interaction (✕, opening the palette, scrolling ≥ 120 px) or 8 s timeout. LocalStorage flag `gibol:search-tooltip-seen` keeps it from re-appearing after dismissal.

### Details

- **Desktop variant**: fixed `top: 56, right: 140`, arrow up, `z-index 48` (below the search palette at 50).
- **Mobile variant**: fixed `bottom: calc(64px + env(safe-area-inset-bottom))`, arrow down, centered horizontally above the "Cari" BottomNav item.
- **Animation**: 240 ms `scale + translateY` pop via a new `@keyframes tooltip-pop` in `src/index.css`.
- **Accessibility**: `role="status"`, `aria-live="polite"`; the ✕ button has `aria-label`.
- **Integration**: dispatch listener for `gibol:open-search` — the same CustomEvent BottomNav and V2TopBar use, so any invocation path dismisses the hint cleanly.

### Why this matters

Audit §04 Persona A (Rangga, mobile user) and §04 Persona B (Diandra, desktop power user) both flagged `⌘K` as gibol's "crown-jewel moment — it should be advertised above the fold." Rangga's version was worse: on mobile there's no keyboard, so even when the pill shows up in the topbar the feature is functionally invisible. v0.11.9 added a bottom-nav search icon; this ship tells new users it exists.

## Verify after deploy

1. **First visit desktop.** Open `www.gibol.co` in an incognito window. 1.5 s after paint, an amber-arrow tooltip appears top-right pointing at the search pill with "Press ⌘K…". Hit `⌘K`, the tooltip dismisses. Reload — it does not come back.
2. **First visit mobile.** Open in incognito on a phone (or Chrome device mode ≤720 px). Tooltip appears at the bottom pointing down at the magnifier icon.
3. **Scroll dismiss.** On a long page, scroll ≥120 px before the tooltip auto-closes. Tooltip disappears; reload does not re-show it.
4. **LocalStorage flag.** Devtools → Application → Local Storage → `https://www.gibol.co` — should see `gibol:search-tooltip-seen` = `"1"` once any dismissal fires.
5. **Reduced motion.** `prefers-reduced-motion: reduce` in DevTools — the pop animation still plays briefly; consider tightening if users complain (cheap follow-up).
