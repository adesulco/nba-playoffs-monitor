# v0.11.2 — Universal V2 masthead across every page

**Date:** 2026-04-23
**Branch:** main
**Risk:** LOW. Pure chrome replacement. No API, no schema, no hook, no data path changed. Every dashboard still fetches and renders the same data below the new bar.

---

## What shipped

### 1. One masthead, everywhere

Before: Home rendered the V2 TopBar (logo · Home / NBA / Football / F1 /
Tennis / World Cup · Search ⌘K · EN · theme · avatar). Every other page —
NBA dashboard, F1, EPL, Tennis, FIFA, Liga 1, Recap, About, Glossary,
pick'em — rendered the legacy v1 TopBar, which had a different shape,
no top-level sport nav, and no search. Users felt a seam flicking
between home and a sport.

After: `V2TopBar` is the single header used on every page. The chrome
looks identical whether you land on `/`, `/nba-playoff-2026`,
`/premier-league-2025-26`, `/tennis`, `/recap`, or `/bracket` — so
hopping between sports feels native.

### 2. Mobile-friendly

`V2TopBar` and `src/index.css` now handle four breakpoints:

- **≤900px** — search pill shrinks to 140px; the ⌘K hint chip hides.
- **≤720px** — search collapses to an icon-only square; nav gains
  horizontal scroll (WebKit scrollbar hidden); padding tightens to 8/12.
- **≤420px** — the decorative avatar dot hides to save space.

The sport-nav row never wraps; it scroll-swipes on touch devices so
"Home · NBA · Football · F1 · Tennis · World Cup" is always reachable
without a hamburger.

### 3. Nothing lost

Every function the v1 TopBar (and NBA's bespoke inline topbar) carried
is preserved:

- Language toggle (EN/BI) — same `toggleLang`, same localStorage key.
- Theme toggle — now via `ThemePopover` (light/dark/auto) instead of
  the two-way sun/moon button.
- Team picker (NBA), constructor picker (F1), club picker (EPL),
  player picker (Tennis) — moved into `V2TopBar`'s new `{children}`
  sub-row.
- NBA live status strip (live-dot · label · WIB time · date · refresh
  age) — same place, now in the sub-row.
- Catatan Playoff deep-link on NBA — preserved in the sub-row.

### 4. V2TopBar API

New `{children}` prop renders as a second row below the masthead,
separated by a hairline border, with `padding: 8px 18px` on desktop
and `8px 12px` on mobile. On mobile the sub-row gains
`overflow-x: auto` so wide content (e.g. TeamPicker chips) can scroll
without wrapping.

No existing caller of V2TopBar breaks — HomeV1 and the new migrations
all pass either nothing or a single `<div>` child.
