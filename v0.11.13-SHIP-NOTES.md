## What shipped

### 1. F1 "News LIVE" concat bug

`src/components/F1News.jsx` — same pattern the v0.11.12 "Drivers Standings2026" fix resolved. The h2 was `"{title}<span>{live chip}</span>"` with a `margin-left: 8` on the span but no actual whitespace node between, so `textContent` flattened to `"F1 NewsLIVE · BAHASA + ENG"`. Added explicit `{' '}` so assistive tech + heading extractors see `"F1 News LIVE · BAHASA + ENG"`. Visual unchanged (the span's margin still draws the gap).

### 2. LiveHero fallback: EPL live when NBA idle

`src/components/LiveHero.jsx` refactored to normalise a sport-agnostic hero struct:

```ts
{ sport, sportLabel, href, clock, home: { abbr, color, score }, away: { ... } }
```

Priority order inside the component:

1. First NBA game with `statusState === 'in'`.
2. Fall-through: first EPL fixture with `statusState === 'in'`.
3. Otherwise → render null (off-day Home unchanged).

Home now calls `useEPLFixtures()` and passes `upcoming` as the `eplFixtures` prop. Tennis / F1 hero fallbacks are deferred — their "is anything live right now" data isn't as cheap to pull on Home mount as EPL's 14-day scoreboard endpoint.

**User-visible effect:** during EPL matchday, when no NBA game is in progress, Home now shows the live EPL fixture in the 72/96/120 px hero (purple/club-accent gradient, "PREMIER LEAGUE · 2025-26" sport stamp, clock in the LIVE chip). Mid-window EPL fans no longer need to click into the sport to find the live score.

### 3. Dead-code janitor

Deleted:

- `src/components/TopBar.jsx` — the v1 masthead, orphaned since v0.11.2 when every page switched to `V2TopBar`.
- `src/components/SportNav.jsx` — the v1 sport-tabs strip, replaced by V2TopBar's primary nav in the same ship.

`CLAUDE.md` flagged both as "leaving in this ship so rollback is a one-line revert per page; delete in a future janitor pass." That follow-up is now done. Grep across `src/` confirms zero imports; production build green.
