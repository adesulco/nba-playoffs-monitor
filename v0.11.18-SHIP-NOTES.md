# v0.11.18 — NBA top gutter + Tennis LiveHero fallback

**Date:** 2026-04-24
**Risk:** LOW. One 16-px spacer on NBA. One new normaliser on LiveHero + two extra tennis-scoreboard fetches on Home. No data path or schema change.

---

## Why

The 2026-04-23 cross-dashboard audit flagged two items we deferred because they needed a design decision:

1. **NBA's first content block sits flush under the V2TopBar subrow**, while EPL/F1/Tennis all use `.dashboard-hero` and get 24 px of breathing room at the top. Flicking from any sport → NBA felt like a 24 px step-up in density.
2. **LiveHero on Home only surfaces NBA or EPL live events.** Tennis Grand Slams can run for 2 weeks with live matches all day; showing the gateway grid on Home during a Wimbledon Monday felt like leaving the mic off.

## What shipped

### 1. NBA content-lead gutter

`src/pages/NBADashboard.jsx` — a `<div style={{ paddingTop: 16 }} />` spacer inserted below the sr-only h1 and above `<YesterdayRecap />`. Gives NBA the "first row doesn't touch the masthead" feel of the other sports without forcing the full 24 px + 20 px bottom of `.dashboard-hero` (which would break YesterdayRecap's intentionally dense chrome).

Chose 16 px over 24 px deliberately: NBA is denser by design (YesterdayRecap + DayScoreboard + 4-col bracket + odds + stories all on one page); 16 is enough to register as breathing room but doesn't push the day-tabs below the fold on smaller desktops.

### 2. Tennis LiveHero fallback

`src/components/LiveHero.jsx` — new `normalizeTennisMatch()` helper maps ESPN's tennis match struct to the sport-agnostic hero struct:

- `players[0]` → `away`, `players[1]` → `home` (ordering matches ESPN competition order)
- Sets won per player → `score` (count of sets where `p1.value > p2.value` or vice versa)
- Last-name 3-letter abbr → `abbr`
- Current set number + round → `clock` (e.g. "SET 3 · QUARTERFINAL")
- Tournament name (uppercase) → `sportLabel` (e.g. "MADRID OPEN")
- Generic `--sport-tennis` accent (#D4A13A) for both sides

`LiveHero` priority order is now **NBA > EPL > Tennis**. F1 deliberately skipped — live F1 windows are ~2h × 23 weekends/year; an always-on Home fetch for that yield isn't worth it. Add when we have a cheap "is a race live right now?" detector (e.g. reading from the already-cached F1 schedule and checking `now within ±2h of race.dateISO`).

`src/pages/Home.jsx` wires:
```js
const { matches: atpMatches } = useTennisScoreboard('atp');
const { matches: wtaMatches } = useTennisScoreboard('wta');
const tennisMatches = useMemo(() => [...atpMatches, ...wtaMatches], [...]);
<LiveHero games={games} eplFixtures={eplFixtures} tennisMatches={tennisMatches} />
```

Two extra fetches on Home, both proxy-cached. The tennis scoreboard hook itself polls 15 s when any match is live / 5 min idle — Home's page weight impact is small.

1. **NBA spacer.** `gibol.co/nba-playoff-2026`. The YESTERDAY recap row should sit ~16 px below the sub-row (TeamPicker strip), not flush against it. Flicking Home → NBA no longer feels like a density cliff.
2. **Tennis LiveHero.** During any live ATP/WTA match when NBA + EPL are both idle: Home's hero should render with the tennis accent, two 3-letter last-name abbrs, sets-won as the 120-px score, and a "SET n · ROUND" clock.
3. **Priority.** During an NBA game: LiveHero stays NBA-themed (tennis doesn't take over).
4. **Off-day.** When nothing anywhere is live: LiveHero returns null, Home shows the gateway grid cleanly (as before).
