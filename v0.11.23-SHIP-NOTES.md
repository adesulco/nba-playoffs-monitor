## What shipped

### GIB-018 — og:title + og:url reflect picker selection
`src/pages/EPL.jsx`, `src/pages/F1.jsx`, `src/pages/Tennis.jsx`.

When a club / constructor / player is picked, the SEO block now leads with that entity:

| Hub | Before | After (with picker) |
|---|---|---|
| EPL | `Liga Inggris 2025-26 · Klasemen 20 Klub …` | `Arsenal · Liga Inggris 2025-26 \| gibol.co` |
| F1 | `Formula 1 2026 · Klasemen Pembalap …` | `McLaren · Formula 1 2026 \| gibol.co` |
| Tennis | `Tenis 2026 · Grand Slam …` | `Carlos Alcaraz · Tenis 2026 \| gibol.co` |

Also: canonical + og:url include the active query string (`?club=arsenal`, `?team=mclaren`, `?player=carlos-alcaraz`) so the share URL matches what the user is actually viewing — not the bare hub URL. WhatsApp / Twitter / Threads unfurls now show topical metadata for deep-linked shares.

**Deferred to a follow-up ship:** per-entity OG PNG generation. Title/url is the high-leverage win for unfurls; per-entity images need a `/api/og/[sport]/[slug].png` generator that's a separate piece of work. The static `og-image.png` keeps serving as the fallback image for now.

### GIB-019 — h1 reflects selected entity
`src/pages/EPL.jsx`, `src/pages/F1.jsx`, `src/pages/Tennis.jsx`. Same pattern at the visible page level so the screen-reader rotor + on-page heading match what the SEO block says.

### GIB-021 + GIB-012 — HomeV1 card density + tap targets
`src/pages/HomeV1.jsx`. The audit's headline was "14.5 px → ≥ 56 px"; three components were under-tap-target:

| Component | Before | After |
|---|---|---|
| `LiveGridCard` cell | padding 10, no minHeight (~55 px when 2 rows present) | padding 14, **minHeight 96**, RowLive crest 16 → 20, name 12 → 13 px, score 13 → 14 px |
| `FollowingCard` row | padding `7px 10px`, crest 18, name 11.5 px (~32 px tall) | padding `10px 10px`, **minHeight 48**, crest 22, name 13 px |
| `UpcomingCard` row | inline link, ~14.5 px tall | **minHeight 44**, padding `8px 8px`, name 12 px, border-radius 6 |

All three now clear WCAG 2.5.5 (Target Size minimum 24×24) and the AAA enhanced 44×44 with margin to spare.

### GIB-024 — Copy link button
**New primitive:** `src/components/CopyLinkButton.jsx`. One-tap copy of the current URL with a 1.6 s inline `Tersalin!` / `Copied!` confirmation, navigator.clipboard with a textarea-execCommand fallback for old Safari, and a `copy_link_click` analytics event tagged with `source` so PostHog can split reads by hub.

**Mounted on:**
- EPL hub — under the lede paragraph, accent `EPL_PURPLE`, source `epl-hub`
- F1 hub — same placement, accent matches the active constructor when picked, source `f1-hub`
- Tennis hub — same placement, accent matches the active player when picked, source `tennis-hub`
- Recap page — inline next to the existing `<ShareButton>`, accent matches the winner color, source `recap`

URL defaults to `window.location.href` so any active picker query (`?club=`, `?team=`, `?player=`) is preserved in the copied URL without any caller-side wiring.

**Why not just open the existing ShareButton popover for copy?** Two clicks (open popover → click "Copy link"). Audit's GIB-024 wanted a one-tap. ShareButton remains the rich path for WhatsApp / X / Threads / IG Story.
