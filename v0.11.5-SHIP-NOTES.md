## What shipped

### 1. `--border-interactive` token ‚Äî WCAG 1.4.11 pass

Added a dedicated token for UI components that need ‚â•3:1 non-text contrast (inactive toolbar chips, inactive search pill, kbd hint). Values:

- Dark: `#5F7390` ‚Äî ~3.7:1 on `--bg` `#0A1628`.
- Light: `#7A7060` ‚Äî ~3.9:1 on `--bg` `#F5F1EA`. (Audit's suggested `#9A8B75` tested at 2.8:1, so it got nudged darker.)

### 2. Picker ARIA + keyboard ‚Äî WCAG 4.1.2

The four custom pickers ‚Äî NBA `TeamPicker`, F1 `ConstructorPicker`, EPL `EPLClubPicker`, Tennis `TennisPlayerPicker` ‚Äî all rendered clickable `<div>` rows inside a `role="listbox"` parent without child roles. Screen-reader users couldn't identify options or navigate; keyboard users had no way to pick a team.

Fix: each row now gets

```jsx
role="option"
aria-selected={isSelected}
tabIndex={0}
onKeyDown={(e) => { if (e.key === 'Enter' || e.key === ' ') { e.preventDefault(); onSelect(‚Ä¶); } }}
```

The "Clear selection" / "Hapus pilihan" row becomes `role="button"` with the same keyboard handler. Full arrow-key navigation + typeahead is a larger piece (Sprint 3) ‚Äî this ship gives VoiceOver / JAWS / NVDA a functional baseline.

Fixes audit P2 "Custom chips missing role='button'."
