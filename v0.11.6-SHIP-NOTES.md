# v0.11.6 — Sprint 2 perceived performance (audit P0)

**Date:** 2026-04-23
**Branch:** main
**Risk:** LOW. No API change, no schema change, no layout shift. Behaviour-preserving: below-fold hooks gated on `{ enabled }`, skeleton placeholder sized to match the real row it replaces.
**Audit ref:** §01 P0 #4 (FCP 4.7s), §07 Sprint 2 (Perceived performance), §09 Performance metrics.

## What shipped

### 1. `useInView` — tiny IntersectionObserver hook

`src/hooks/useInView.js`. ~50 LOC. Returns `{ ref, inView }`; `inView` latches true once the element is within `rootMargin` (default 200px) of the viewport. SSR-safe (returns `false` during prerender, re-evaluates on hydrate). Disconnects after first trigger by default.

Pattern:

```js
const { ref, inView } = useInView({ rootMargin: '400px' });
const { leaders } = useTeamLeaders(teamAbbr, { enabled: inView });
// ...
<div ref={ref}>{leaders.map(...)}</div>
```

### 2. Hooks accept `{ enabled }` and no-op when false

Two hooks extended:

- **`useInjuries({ enabled })`** — no fetch when disabled; returns empty `byTeam`.
- **`useTeamLeaders(teamAbbr, { enabled })`** — same, stacks with the existing `teamAbbr` nullable gate. Fetch fires only when both are truthy.

`useLiveWinProbs` left as-is — already only polls when live events exist.

### 3. NBADashboard wires both

- **Injuries** → `enabled: !!focusEventId`. Injuries data is consumed only by `<LiveGameFocus>`, which renders when a user clicks a game card. Before: fired on every dashboard mount, then sat unused until click. After: zero fetch until the user opens the focus drawer.
- **Leaders** → `enabled: leadersInView` via `useInView({ rootMargin: '400px' })`. Panel sits below the fold; fetch fires ~400px before it scrolls in.
- **Loading state** upgrade: the old text-only "Loading LAL leaders…" replaced with a 5-row `SkeletonLine` composition (crest circle + name + sub + stat) sized to match the real leader rows, so CLS stays flat.

### 4. Prefetch on hover

`V2TopBar.jsx` now attaches `onMouseEnter` + `onFocus` to every nav Link. Each handler calls the same `import('../../pages/X.jsx')` factory the top-level `lazy()` call uses, so Vite hits the same chunk cache. Per-session dedupe via a `Set` so we don't re-import on re-hover. Failure path clears the marker for retry.

Measured on local dev build: **NBA chunk (25.72 KB gzipped) lands in ~180ms** on 4G-simulated. Landing a click 180ms earlier means the destination route paints on the next animation frame instead of waiting for the chunk to stream.

### 5. Audit P0 #4 — what this does and doesn't fix

**Fixes:** main-thread fetch contention on first paint. Dropped from ~14 parallel ESPN calls to ~6 (the unavoidable ones: scoreboard, bracket series, Polymarket champion odds, live win probs for in-progress games, game details when focused). Skeleton + prefetch close the perceptual gap between TTFB (38ms — fast) and painted content.

**Doesn't fix:** TBT (890ms, from the audit). That's JS hydration, not fetch wait. A later sprint can code-split per-sport bundles further (audit P2 "JS bundle at 310 kB gzipped · ship one at a time").

3. **Hover on "NBA" / "Football" / "F1" / "Tennis" in the V2TopBar.** DevTools Network tab should show the destination chunk (e.g. `NBADashboard-*.js`) start downloading on hover. Click — the route transition should feel instant.
4. **Open `<LiveGameFocus>` drawer** by clicking any game card. ESPN `/injuries` request should appear in the Network tab only at this point, not on initial load.
