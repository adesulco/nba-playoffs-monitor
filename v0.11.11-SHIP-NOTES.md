## What shipped

### 1. `<LiveHero />` — editorial live-game hero on Home

`src/components/LiveHero.jsx`, ~180 LOC. Picks the first in-progress NBA game from `games` and renders a full-width hero above the Milikmu strip + dashboard grid.

Layout:

```
● LIVE · Q3 4:12                         NBA PLAYOFF 2026
┌────────────────────────────────────────────────────────┐
│                                                        │
│  [OKC]  OKC                                       98   │
│  [LAL]  LAL                                       94   │
│                                                        │
│  ● Skor live · buka dashboard →                        │
└────────────────────────────────────────────────────────┘
```

- **Typography:** score numerals are responsive tabular Inter Tight 800 — 72 px on mobile, 96 px ≥720 px, 120 px ≥1100 px. Tabular-nums so the digits line up vertically between teams. Letter-spacing -2 for the editorial tightness the audit §05 spec calls out.
- **Signature tick device:** 3 px gradient strip across the top edge (live red → amber → transparent). Static; animation reserved for prefers-reduced-motion-respecting future work.
- **Team identity:** 36×36 team accent tile + 3-letter abbr. Winner = `--amber-2` + bold; trailing = `--ink-3`. Mirrors the .home-card-live hover-lift treatment so it reads as a single clickable destination.
- **Clickable** — wraps `<Link to="/nba-playoff-2026">` with an `aria-label` announcing the live score for screen readers.
- **Null-safe** — renders nothing when no NBA games are in-progress, so off-day Home keeps its existing gateway shape.

### 2. `.tick-live` utility — signature amber tick device

`src/index.css`. A 28×2 amber underscore via `::after` on whatever element you apply the class to. Zero layout impact on untouched elements; element gains 8 px of bottom padding so the tick sits clear of the text baseline.

Applied to:

- **EPL `<LiveSpotlight>` h2** — "Sedang main" / "Live now" now carries the signature mark whenever EPL has live matches.
- **NBA `DayScoreboard` "SCORES & SCHEDULE" header** — conditional; the tick renders only when `activeCount > 0` (i.e. the active day has live games), so on off-days the heading stays clean.

### 3. What's NOT in this ship

- **Type-scale expansion (4 → 7 steps)** — the audit's Visual Design finding. We actually already have 8 CSS size tokens in `src/index.css` (`--size-caption`/`--size-label`/…/`--size-display-lg`); the gap is that components use hardcoded `fontSize: 20` etc. Tightening that is mechanical but touches ~40 files — deferring to a dedicated polish ship rather than racing it alongside identity work.
- **Pre-roll animation on the live tick** — the audit described "subtly animated." I built a static gradient here because a motion treatment that reads well on both light + dark backgrounds needs design review; I don't want to ship a hero animation that only looks right on dark. Slotted for Sprint 6 motion pass.
