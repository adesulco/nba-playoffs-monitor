### 2. v2 redesign Phase 1 foundation (invisible — behind flag)

Tokens already align with v2 spec from the Brand v1 refresh in v0.3.0, so no CSS work needed. Added:

- **[`src/lib/flags.js`](src/lib/flags.js)** gains a `UI` namespace with 9 flags — `ui_v2`, `cmdCenter`, `terminalHome`, `pickem`, `xgPerShot`, `tennisPbp`, `f1TyrePit`, `wc2026Teaser`, `liga1Teaser` — all defaulting `false`. Env override via `VITE_FLAG_UI_V2=1`, `VITE_FLAG_CMD_CENTER=1`, etc. Coexists with existing `VISIBLE`/`LIVE` namespaces; no existing flag renamed or removed.

- **[`src/lib/AppContext.jsx`](src/lib/AppContext.jsx)** extended:
  - `theme` accepts a third value `'auto'` (follows OS, live-updates via `matchMedia` listener).
  - `effectiveTheme` derived — always `'dark' | 'light'` — for callers that need the resolved value.
  - New `setTheme(next)` method validates `'auto' | 'dark' | 'light'` for the v2 top-bar popover.
  - `toggleTheme()` preserved for back-compat.
  - New `accent` state + `setAccent(hex | null)` bridges per-user hex to the `--accent` CSS var (validated as `/^#[0-9A-F]{6}$/i`).
  - `lang` internal storage keeps `'en' | 'id'`. `'bi'` accepted as a read synonym mapped to `'id'` so the v2 UI can label the toggle "BI" without invalidating any existing user's stored pref. `<html lang>` stays ISO-639-1 compliant.

- **[`src/lib/i18n.js`](src/lib/i18n.js)** gains 34 v2 keys in both `en` and `id` blocks (home, liveNow, followingNav, addToBoard, multiMatch, aiRecap, championshipOdds, topPerformers, momentum, bracket, standings, fixtures, scheduleOfPlay, todayReaction, west, east, serving, aces, dblFaults, firstServe, breakPts, pitWindow, tyreStrategy, commentary + share-card + state + theme-popover labels). BI copy is casual register per the voice guide. No existing key renamed or removed.

### 3. Vocabulary guard (CI)

**New** [`scripts/check-vocab.mjs`](scripts/check-vocab.mjs) greps `src/**/*.{js,jsx,ts,tsx,mjs,cjs}` for forbidden betting/gambling vocabulary (`taruhan`, `bertaruh`, `judi`, `bet`, `bets`, `betting`, `gamble`, `gambling`, `pasang taruhan/judi/bet`). Runs as the first step of `npm run build` so regressions cannot ship to production.

Rule source: [`docs/v2-design-gaps.md §D8`](../docs/v2-design-gaps.md). Indonesian gambling law (KUHP 303, UU 7/1974) + editorial policy forbid framing Pick'em, odds, or any feature as betting. Odds display as market-signal editorial (Polymarket, moneylines) is allowed — but never as "bet this" or affiliate-linked.

Tested both directions: passes on clean source, fails on regression with exit code 1. `npm run check:vocab` exposes it standalone.

### 4. Glossary copy scrub

`src/pages/Glossary.jsx:13` — "Peluang juara" definition: *"pergerakan taruhan"* → *"pergerakan pasar"*. The odds movement is driven by the prediction market itself, not wagers. Only user-facing use of the banned vocabulary anywhere in `src/`.
