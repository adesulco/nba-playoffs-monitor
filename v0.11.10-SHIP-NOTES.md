# v0.11.10 — Sprint 4 week-2: score-change flash + haptic

**Date:** 2026-04-23
**Branch:** main
**Risk:** LOW. Additive effect only, inert outside live games; no data path or layout change. One CSS keyframe + two small `useEffect` blocks in existing score-card components.
**Audit ref:** §05 stretch spec — "Score changes animate (tiny scale + 300 ms colour flash) — matches the emotional beat of sport."

---

## What shipped

### 1. `.score-flash` CSS keyframe

`src/index.css`. 600 ms one-shot animation:

```
0%   → scale(1),     no glow,  inherit color
20%  → scale(1.15),  16px amber glow, color: var(--amber-2)
100% → scale(1),     no glow,  inherit color
```

Applied as a className; auto-clears via the `setTimeout(..., 600)` in the consumer. Honours `prefers-reduced-motion` — users who've opted out get a color-only pivot with no scale/glow.

We went 600 ms instead of the audit's 300 ms because 300 ms read as a glitch during local testing — eyes barely caught the change. Can be dialed back via the keyframe alone if field data disagrees.

### 2. NBA DayScoreboard GameCard

Per-side `useRef` tracks previous `away.score` / `home.score`. A `useEffect` on `[awayScore, homeScore, isLive, favInGame]`:

- Guards on `isLive` so finals / upcoming don't fire on first mount.
- Detects delta against the prev ref.
- Sets a local `awayFlash` / `homeFlash` state true → CSS class applies → 600 ms timeout clears it.
- Fires `navigator.vibrate(40)` **only when the user's favourite team is in the game** — a user watching the whole league shouldn't have their phone buzz eight times per night.
- Updates the prev ref either way so the next tick sees the fresh baseline.

### 3. EPL DayScoreboard MatchCard

Same pattern, one difference: EPL renders a single combined "home – away" score line, so one `flash` flag drives the whole element. No per-user favourite gate on the vibrate yet (EPL's `selectedEPLClub` is orthogonal to "is my club in this match"); straightforward follow-up when we wire that.

## Verify after deploy

1. **During live play** (NBA or EPL). Wait for a basket / goal. The score number should briefly swell amber for ~600 ms, then settle. On a phone with the user's fav team in the game, a 40 ms buzz fires on the same frame.
2. **First paint doesn't flash.** Refresh during a live game — the initial render of the current score should NOT animate. Only subsequent deltas trigger the flash.
3. **Finals / upcoming stay still.** Open a finished game card on a day-tab. The score is static; scroll away and back — no stray flash.
4. **Reduced motion.** DevTools → Rendering → "Emulate CSS prefers-reduced-motion: reduce." Score change pivots color only — no scale or glow pulse.
